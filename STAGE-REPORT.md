## 一、项目概况

### 1.1 项目起源与命名

Mayfly项目脱胎于此前未落地的「虫音」项目，延续了初心打造技术价值的核心初衷。项目名称寓意即便自身如蜉蝣般普通，也能凭借技术产品发出自身声音、实现破局成长。

项目由个人独立开发，依托AI辅助完成MVP落地，是聚焦Spring AI生态的企业级大模型调用治理工具，也是**当前Spring AI生态内首个覆盖完整企业级模型治理能力的开源项目**。

### 1.2 项目基本信息

| 项目属性 | 内容 |
|---------|------|
| 项目名称 | Mayfly |
| 项目定位 | 基于Spring AI的企业级模型路由增强插件 |
| 核心价值 | 让每个使用Spring AI的Java开发者零成本获得企业级模型路由能力 |
| 技术定位 | 「Spring AI生态的专属Spring Cloud」 |
| 技术栈 | Java 17+ / Spring Boot 3.2+ / Spring AI 1.0+ |
| 许可证 | Apache License 2.0 |
| 开发模式 | 全AI Agent团队开发 |

## 二、技术生态关系

### 2.1 层级架构

Mayfly与Spring Boot、Spring Cloud、Spring AI形成**逐层依赖、互补增强、非互斥**的层级关系：

| 层级 | 框架 | 职责 |
|------|------|------|
| 基础层 | Spring Boot | 业务代码开发、自动配置、应用运行 |
| 治理层 | Spring Cloud | 微服务注册、负载均衡、熔断降级 |
| AI层 | Spring AI | 统一大模型调用接口、基础AI能力集成 |
| 治理增强层 | **Mayfly** | 大模型专属治理（路由、负载、熔断、监控） |

### 2.2 与Spring AI Alibaba的关系

**完全互补，无直接竞争**：
- Spring AI Alibaba聚焦AI智能体、工作流编排
- Mayfly聚焦大模型调用底层治理
- 企业可同时使用两者，共同完善Spring AI企业级生态
